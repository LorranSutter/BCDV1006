Lab 1.3: Intro CSS and the box model

How to calculate the width and height of an element using the box-model?

In CSS the box-model have four different parts: margin, border, padding and content. Each layer wrapped by the previous one starting with margin.

The first three parts have top, right, bottom and left thickness. And content have only width and height.

As long as margin layer do not belongs to the element, its thickness is not considered to calculate element width and height (source: w3schools).

So, to calculate the total width of the element you must sum the width of the content with the left and right thickness of padding and border.

To calculate the height, you must do the same process, however sum top and bottom thickness with the content height.

Funny element

div {
 border: 1px solid purple;
 box-sizing: border-box;
 height: 250px;
 margin: 5px 3px 2px 7px;
 padding: 4px 3px 2px;
 width: 400px;
}
div {
 border: 1px solid purple;
 box-sizing: content-box;
 height: 250px;
 margin: 5px 3px 2px 7px;
 padding: 4px 3px 2px;
 width: 400px;
}

What is the total height and total width of the element?

  • Element height: 258px
  • Element width: 408px

What is the height and width of the content area?

  • Height content: 250px
  • Width content: 400px

The Position attribute

  1. What does position absolute do?
    • In this case, the element is removed from the document flow and it is positioned based on its parent element. So, if its left atribute is 0, its position will start in the top left of its parent element affecting the behave of the others elements. (source frecodecamp)
  2. What does position relative do?
    • Relative position is the default for this attribute. It positioned based on its normal position. If you set its left attribute to 50px, it will me moved 50px from the last closet element. (source freecodecamp)
  3. What does position fixed do?
    • If you have, for instance, a figure with the attribute fixed it will be displayed in your browser regardless of where you scrolled your page. It will overlap any other element with other position attribute, because it is positioned relative to your browser. (source w3schools)
